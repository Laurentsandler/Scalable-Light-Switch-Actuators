# 2/20/2026 11 AM - Building the Concept and Setting Goals

_Time spent: 1.0h_

I have a problem which I am sure many other people do. I live in a rented out house, so we cannot install our own smart light switches. And the only other solution on the market is actuators that you put on the outside that move your light switch with remote. And they are very, very bulky. And don't support smart home like Amazon Alexa, Home Assistant, Zigbee, etc. They only use a remote. So I've decided to build my own. Using a custom PCB. Here are some goals:

- small and easy to install
- long battery life
- works with apple homekit, and blynk
- easy to charge with a charging dock
- User is still able to use light switch through a button on the device
- cost effective
- uses esp-now to connect switches together in no wifi areas of the home

Once I had these goals set, I made some sketches of how it would work and how it would look.

There are two versions which I came up with on how the PCB and server should be located. One where the servo sits on the PCB and the battery on top and another one where the board takes up the back of the servo, and the servo sits on it, and then the battery is off to the side.

I chose to do a combination of both, where I have the PCB take up only the footprint of the servo, and the servo will sit on the PCB. Then the battery will sit on the PCB too and take up the same footprint and be off to the side.

# 2/20/2026 12 PM - Choosing an ESP32 module

_Time spent: 0.3h_

I eventually settled on the ESP32-C3-WROOM-02 because:

- **Low power**: Efficient RISC-V core, ~5 µA deep sleep, ideal for wake → move servo → sleep cycles.
- **Compact**: ~18 × 20 mm module, fits easily under/behind a servo.
- **Wi-Fi + BLE built-in**: Supports ESP-NOW; BLE enables HomeKit (via ESPHome/Matter), provisioning, and future Matter support.
- **Cost-effective**: Cheaper than S3 and significantly less than WROVER modules.

# 3/1/2026 - Writing the code

_Time spent: 2.5h_

Today I wrote the firmware for each of the modules and the ESP32 base station.

The base ESP32 connects to WiFi/Blynk and talks to 5 ESP32-C3 modules over ESP-NOW. When you press a Blynk button (V0–V4), the base sends a toggle command to the matching module, which moves its servo ±20° to flip a light switch.

**To set it up:**

1. Flash each C3 module with the corresponding "Module" code and note the MAC address it prints on serial
2. Paste those MACs into the "base code" along with your WiFi and Blynk credentials
3. Flash the base ESP32 with "Base" code
4. In Blynk, create 5 button widgets on V0–V4 (Push mode, Integer 0/1)
